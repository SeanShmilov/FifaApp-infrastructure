# 🚀 How to Restart FraudstersList Infrastructure

Because you have successfully automated your infrastructure using **Terraform**, **Kubernetes**, and **GitOps**, spinning everything back up from scratch is incredibly simple. 

## Step 1: Build the Infrastructure (Terraform)
Terraform is responsible for building your "physical" cloud components.
1. Go to **Terraform Cloud**.
2. Click **New Run** -> **Choose Plan and Apply**.
3. Wait about 10–15 minutes. 
   - *Behind the scenes, Terraform will automatically create your VPC, EKS Cluster, Node Groups, the AWS Load Balancer Controller, and install the base ArgoCD software!*

## Step 2: Connect Your Terminal
Once Terraform finishes, you need to tell your local terminal how to communicate with your brand-new EKS cluster.
1. Open your terminal in the `FraudstersList-infrastructure` folder.
2. Run this command to download the cluster credentials:
   ```bash
   aws eks update-kubeconfig --region us-east-1 --name fraudsterslist-eks
   ```

## Step 3: Inject your Database Secret via CLI
Because we upgraded to AWS Secrets Manager, you don't use `kubectl create secret` anymore! Instead, you inject the password directly into your AWS Vault using the AWS CLI.

1. Run this command in your terminal, replacing the password with your real database password:
   ```bash
   aws secretsmanager put-secret-value \
     --region us-east-1 \
     --secret-id fraudsterslist/mongo_uri \
     --secret-string "mongodb+srv://fifa_user:<YOUR_REAL_PASSWORD>@cluster0.j32jife.mongodb.net/fraudsterslist?retryWrites=true&w=majority"
   ```

## Step 4: Give ArgoCD its Assignment
Terraform automatically installs ArgoCD, but ArgoCD boots up empty. You need to give it its "assignment" so it knows which GitHub repository to watch.
1. Apply the ArgoCD Application configuration:
   ```bash
   kubectl apply -f argocd-app.yaml
   ```
2. *As soon as you run this command, ArgoCD will instantly read your `k8s/` folder and start deploying your frontend and backend!*

## Step 5: Login to ArgoCD
To access the ArgoCD UI and watch your pods spin up:
1. Retrieve your autogenerated ArgoCD admin password by running:
   ```bash
   kubectl -n argocd get secret argocd-initial-admin-secret -o jsonpath="{.data.password}" | base64 -d && echo
   ```
2. Find your Load Balancer URL by running:
   ```bash
   kubectl get ingress -A
   ```
3. Open your browser and go to `http://<YOUR-LOAD-BALANCER-URL>/argocd/`.
4. Login with username `admin` and the password you just retrieved.

## Step 6: View the Live Website
Once all the pods in ArgoCD turn **Green (Healthy)**, simply remove `/argocd/` from the end of the URL. 
- Go to `http://<YOUR-LOAD-BALANCER-URL>` to see your live FraudstersList!

## Step 7: View Grafana Dashboards
To view your cluster metrics, navigate to the Grafana dashboard:
1. Go to `http://<YOUR-LOAD-BALANCER-URL>/grafana/`.
2. Login with username `admin` and password `prom-operator`.
***

## 🛑 How to Safely Destroy (To Save Money)
Because ArgoCD automatically creates physical AWS Load Balancers for your apps *outside* of Terraform's control, you must tell ArgoCD to delete your apps **before** you trigger the Terraform Destroy, otherwise Terraform will get blocked by the Load Balancer subnets!

1. Open your terminal in the `FraudstersList-infrastructure` folder.
2. Run this command to wipe the app:
   ```bash
   kubectl delete application fraudsterslist -n argocd
   ```
3. Run these commands to wipe all Ingresses (which triggers the physical Load Balancer deletion):
   ```bash
   kubectl delete ingress argocd-ingress -n argocd
   kubectl delete ingress fraudsterslist-ingress -n fraudsterslist
   kubectl delete ingress grafana-ingress -n monitoring
   ```
4. Wait exactly **2 minutes** for AWS to delete the physical load balancers.
5. Go to Terraform Cloud and click **Queue Destroy Plan**. It will now tear down cleanly!
